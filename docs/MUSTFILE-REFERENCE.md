# Mustfile Reference

## What Is a Mustfile?

A **Mustfile** is a declarative contract of **Physical State** — the verifiable,
observable condition of project artifacts. It defines what _must_ be true about
a repository before commits, pushes, CI runs, or releases proceed.

The relationship is: **Mustfile:must :: Justfile:just**

- The **Mustfile format** is specified in
  [`contractiles/mustfile/`](../contractiles/mustfile/) (this repo).
- The **`must` binary** that executes Mustfiles is implemented in Ada 2022 at
  [hyperpolymath/must](https://github.com/hyperpolymath/must).

## What `must check` Does

`must check` reads a Mustfile and evaluates each declared check against the
current repository state. Each check has a severity level:

| Severity | Behaviour |
|----------|-----------|
| `BLOCK`  | Fails the build/commit if the check fails |
| `WARN`   | Continues with a warning message |
| `INFO`   | Logs only, no failure |

Checks can verify:
- **File presence/absence** — files that must or must not exist
- **Content requirements** — strings that must appear in specific files
- **Commands** — arbitrary shell commands that must succeed (formatting, linting, tests)
- **Schema versions** — metadata version constraints
- **Artifacts** — compiled binaries, container images

### Example Mustfile

```yaml
# SPDX-License-Identifier: PMPL-1.0-or-later
version: 1

checks:
  - name: security
    run: just lint
  - name: tests
    run: just test
  - name: format
    run: just fmt
```

### Lifecycle Hooks

Checks are grouped by lifecycle phase:

| Phase | When It Runs |
|-------|-------------|
| `[pre-commit]` | Before `git commit` (via git hooks) |
| `[pre-push]` | Before `git push` |
| `[ci]` | During CI pipeline execution |
| `[release]` | Before tagging a release |

## How Mustfile Relates to the Contractile CLI

The Mustfile is one of **five contractile types** in the contractile CLI system:

| Type | File | CLI Command | Purpose |
|------|------|------------|---------|
| **Must** | `Mustfile.a2ml` | `must check \| fix \| enforce \| list` | Physical State — what must be true |
| **Trust** | `Trustfile.a2ml` | `trust verify \| hash \| sign \| list` | Provenance State — what to verify |
| **Dust** | `Dustfile.a2ml` | `dust status \| rollback \| replay \| run` | Reversibility State — how to recover |
| **Intend** | `Intentfile.a2ml` | `intend list \| check \| progress` | Intent State — roadmap commitments |
| **K9** | `*.k9.ncl` | `k9 eval \| run \| typecheck \| info` | Self-validating Nickel components |

Together, these five types form the **Contractile Stack** — a file-first
contract system for operating, verifying, recovering, and evolving repositories.

### The Contractile Stack Architecture

```
┌─────────────────────────────────────────────────────────┐
│                      Justfile                           │
│   import "contractile.just"    <- auto-generated        │
│   just must-check, just trust-verify, just dust-status  │
├─────────────────────────────────────────────────────────┤
│                  contractile.just                        │
│   Generated by: contractile gen-just                    │
├──────┬──────┬──────┬───────┬────────────────────────────┤
│ must │trust │ dust │intend │ k9        <- CLI binaries   │
├──────┴──────┴──────┴───────┴────────────────────────────┤
│            contractile-core (Rust library)               │
│   A2ML Parser  |  K9 Bridge  |  Just Emitter            │
├─────────────────────────────────────────────────────────┤
│         *.a2ml files          |  *.k9.ncl files          │
│   (declarative contracts)    |  (validated components)   │
└─────────────────────────────────────────────────────────┘
```

## How the RSR Template Provides a Mustfile

The [rsr-template-repo](https://github.com/hyperpolymath/rsr-template-repo)
includes:

1. **`Justfile`** — imports `contractile.just` via `import? "contractile.just"`,
   which auto-generates Just recipes for all five contractile types.
2. **`contractile.just`** — generated by `contractile gen-just`, provides
   `must-check`, `trust-verify`, `dust-status`, `intend-list`, and `k9-eval`
   recipes.
3. **Template contractile files** in `docs/templates/contractiles/` and
   `.machine_readable/contractiles/` for bootstrapping new repos.

When you run `just init` from the template, placeholders are replaced and the
contractile infrastructure is ready to use.

## Quick Start

```bash
# Install the contractile CLI (from this repo)
just build-cli
just install-cli

# Generate Just recipes from your contractile files
contractile gen-just

# Run must checks
must check            # or: just must-check

# Run all contractile checks
just contractile-all  # must + trust + k9
```
