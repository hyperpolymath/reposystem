= Intentfile — Future Intent & Roadmap Contract
:author: Jonathan D.A. Jewell

// SPDX-License-Identifier: PMPL-1.0-or-later

An Intentfile declares **what we commit to making true** — architectural direction, planned features, migration roadmap, and compliance goals. It is the Intent State contract.

The directory name `lust/` is a legacy naming convention (future intent). The CLI accepts both `contractiles/lust/` and `contractiles/intend/`.

== Canonical File

`Intentfile.a2ml` (this directory)

== Quick Start

Copy `Intentfile.a2ml` into your project at `contractiles/lust/Intentfile.a2ml`, then edit to declare your project's roadmap.

== Format

Intentfiles support two styles:

=== Simple Intents (Prose)

For lightweight intent tracking:

[source,a2ml]
----
## Features

- add async support for all public APIs
- support no_std environments
----

=== Structured Intents (Full Metadata)

For tracked intents with lifecycle, priority, and evidence probes:

[source,a2ml]
----
## Features

### async-support
- description: Add async support for all public APIs
- status: in-progress
- priority: high
- target: 2026-Q3
- owner: core-team
- evidence: command: grep -q 'async' src/lib.rs
- depends_on: runtime-refactor
- notes: Blocked on tokio 2.0 release
----

== Intent Lifecycle

Every structured intent moves through:

[source]
----
declared → accepted → in-progress → realised
                                        ↘ abandoned
                                   superseded
----

The CLI enforces valid transitions:

[source,bash]
----
intend accept NAME           # declared → accepted
intend start NAME            # accepted → in-progress
intend realise NAME          # in-progress → realised
intend abandon NAME --reason "..."  # any → abandoned
intend supersede OLD NEW     # any → superseded
----

== Evidence Probes

The `evidence:` field declares how `intend check` detects realisation:

[cols="1,3"]
|===
| Format | Meaning

| `FILE exists` | Check file existence
| `FILE contains PATTERN` | Check file contains string
| `command: CMD` | Run shell command (exit 0 = realised)
| `must: CHECK_NAME` | Delegate to a must check
| `trust: VERIFY_NAME` | Delegate to a trust verification
|===

== CLI Commands

[source,bash]
----
intend list             # Display roadmap with status indicators
intend check            # Run evidence probes
intend check --json     # JSON output for CI
intend progress         # Summary by lifecycle status

# Lifecycle transitions (modify Intentfile.a2ml in place):
intend accept NAME
intend start NAME
intend realise NAME [--note "..."]
intend abandon NAME --reason "..."
intend supersede OLD NEW
----

== Status Indicators

When displayed with `intend list`:

[cols="1,3"]
|===
| Symbol | Status

| `[ ]` | declared (not yet committed to)
| `[+]` | accepted (committed, not started)
| `[~]` | in-progress (active work underway)
| `[x]` | realised (done and verified)
| `[/]` | abandoned or superseded
|===
