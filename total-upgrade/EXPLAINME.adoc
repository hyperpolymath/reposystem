This file provides code-level evidence and implementation details for the claims made in README.adoc.

== Claim-to-implementation map

=== Manager Detection

[quote, README.adoc]
____
Real-time detection of asdf, mise, and opsm.
____

How this is implemented::
The `Detector` struct in `src/backend/detector.rs` executes `which <tool>` and `<tool> --version` using `std::process::Command`. 

Caveat::
Depends on the tools being in the system `PATH` at the time of execution.

=== Manifest Integration

[quote, README.adoc]
____
Unified view of tools managed by .tool-versions.
____

How this is implemented::
The `ManifestParser` in `src/backend/manifest.rs` performs a targeted read of `~/.tool-versions`, parsing it into a `HashMap<String, String>` which is then displayed in the TUI.

Caveat::
Currently only parses the global `~/.tool-versions`. Support for local project files and `mise.toml` is planned.

=== Proactive Discovery

[quote, README.adoc]
____
Scans for missing package managers when a runtime (Python, Node, etc.) is found.
____

How this is implemented::
`Detector::discover_ecosystems` in `src/backend/detector.rs` checks for specific runtime/package-manager pairs (e.g., `ruby` -> `gem`) and returns a `DiscoveryStatus`.

Caveat::
Hardcoded list of common ecosystems. Extending this requires updating the `checks` vector in `detector.rs`.

=== Association Disambiguation

[quote, README.adoc]
____
Handles complex file associations like Coq vs V-lang .v files.
____

How this is implemented::
`Scanner::scan_associations` in `src/backend/scanner.rs` checks for the presence of multiple possible tool chains for the same extension and reports ambiguity if multiple are found.

Caveat::
Currently relies on tool presence in `PATH`. Does not yet scan project file markers ( e.g. `_CoqProject`).

== Architecture & Unification

TotalUpgrade follows the account-wide standards for inter-service communication.

=== The Groove/Cleave Paradigm

[IMPORTANT]
====
**The Groove Protocol** provides the **"How we talk"** (communication, capability negotiation, and eventing). +
**The Cleave Surface** defines the **"How we touch"** (physical integration, binary layout, and formal verification of the join/split point).
====

TotalUpgrade currently utilizes **Soft Groove**...

== Known Gaps

[CAUTION]
====
Mise.toml: README claims support for mise, but current implementation only covers .tool-versions.
====

[CAUTION]
====
Transfer Action: The UI allows selecting tools for transfer, but the write-back logic to manifest files is not yet implemented.
====
