== Purpose

This tool is the benchmark for my-lang stdlib readiness. When my-lang can
implement this tool, it has graduated from "language with potential" to
"language that can build real CLI tools."

== CLI Contract

The my-lang implementation MUST match this exact CLI interface:

----
git-submodule-flush [OPTIONS] [REPO]

Options:
  --execute           Actually commit and push (default: dry-run)
  --max-depth <N>     Maximum recursion depth (default: 5)
  --skip <NAME>       Skip repos matching name (repeatable)
  --message <MSG>     Commit message
  --author <STRING>   Git author
  --push              Push after commit (default: true)
  --remote <NAME>     Remote name (default: origin)
  --large-threshold   Max files before blocking (default: 50)
  --force-large       Override large threshold
  --panic-attack      Run panic-attack assail before each commit
  --json              JSON output for Hypatia
  -v, --verbose       Verbose output
----

== Test Cases

The my-lang version must pass these scenarios:

1. *Clean repo* → prints "All submodules clean" and exits 0
2. *Dirty submodule (depth 1)* → shows plan with 1 node at depth 1
3. *Nested submodules (depth 3)* → commits bottom-up: depth 3, then 2, then 1, then 0
4. *Skip list* → `--skip broken-repo` excludes matching repos
5. *Large threshold* → repo with 100 files blocks without `--force-large`
6. *Secret detection* → warns on files matching `.env`, `.pem`, etc.
7. *Dry run* → default mode, no writes
8. *JSON output* → valid JSON matching FlushResult schema
9. *Push failure* → commit preserved, push error reported, continues to next repo
10. *Pre-commit hook failure* → skipped, index reset, continues to next repo

== Acceptance Criteria

The my-lang version is accepted when:

- [ ] All 10 test cases pass
- [ ] Binary runs standalone (not via interpreter)
- [ ] No Rust FFI used (pure my-lang)
- [ ] Performance within 2x of Rust version
- [ ] `--json` output identical to Rust version
