Contractiles is a file-first contract system for operating, verifying, recovering, and evolving repositories. It centers on five portable contract artifacts — Mustfile, Trustfile, Dustfile, Intentfile, and K9 — that you copy into any repo to make runtime behavior, integrity, recovery, and roadmap explicit and testable.

Version: v1.1.0

== The Five Contractile Types

[cols="1,2,2,2"]
|===
| Type | File | CLI | Purpose

| *Must*
| `Mustfile.a2ml`
| `must check \| fix \| enforce \| list`
| Physical State — what must be true (files, configs, invariants)

| *Trust*
| `Trustfile.a2ml`
| `trust verify \| hash \| sign \| list`
| Provenance State — what to verify (hashes, signatures, proofs)

| *Dust*
| `Dustfile.a2ml`
| `dust status \| rollback \| replay \| run`
| Reversibility State — how to recover (rollback, undo, replay)

| *Intend*
| `Intentfile.a2ml`
| `intend list \| check \| progress`
| Intent State — what we commit to making true (roadmap, migrations)

| *K9*
| `*.k9.ncl`
| `k9 eval \| run \| typecheck \| info`
| Self-validating Nickel components (Kennel/Yard/Hunt levels)
|===

== Quick Start

[source,bash]
----
# Install the CLI
just build-cli
just install-cli

# Generate Just recipes from A2ML + K9 sources
just gen-just

# Run all checks
just check          # must checks (Physical State)
just verify         # trust verifications (integrity)
just audit          # check + verify combined

# Use CLIs directly
must check          # run all Mustfile checks
trust verify        # run all Trustfile verifications
dust status         # list available recovery actions
intend list         # display roadmap with status
intend check        # run evidence probes

# Or via Just recipes
just must-check
just trust-verify
just dust-status
just intend-list
----

== Format Policy (A2ML Canonical)

* Canonical contract format is **A2ML** (Annotated Markup with Machine Logic)
* All contracts live in `contractiles/`:
** `contractiles/must/Mustfile.a2ml`
** `contractiles/trust/Trustfile.a2ml`
** `contractiles/dust/Dustfile.a2ml`
** `contractiles/lust/Intentfile.a2ml`
* Legacy files (`Mustfile`, `Trustfile.hs`, `Dustfile`, `Intentfile`) are deprecated compatibility shims
* K9 components use Nickel (`.k9.ncl`) with three security levels (The Leash)

== How A2ML and K9 Cooperate

[cols="1,3"]
|===
| Layer | Role

| *Spec* (`.adoc`)
| Defines grammar, semantics, exit codes

| *A2ML* (`.a2ml`)
| Carries the declaration (human-readable, line-oriented)

| *K9* (`.k9.ncl`)
| Validates and extends (Nickel type contracts, Hunt-level execution)

| *Runner* (`must`/`trust`/etc.)
| Parses A2ML, executes commands, reports results

| *Just* (`.just`)
| Integrates into repo workflow via generated recipes
|===

* A K9 Yard component can **validate** an A2ML file's semantics beyond what the parser checks
* A K9 Hunt component can **implement** a contractile action with type-safe config
* The runner CLI is the bridge — reads A2ML, optionally delegates to K9, emits to Just

== Usage Model

1. Copy the `contractiles/` directory into your repo
2. Edit the `.a2ml` files to declare your project's contracts
3. Run `contractile gen-just` to generate Just recipes
4. Add `import "contractile.just"` to your Justfile
5. Run `just check` / `just verify` in your CI/CD pipeline
6. Use `intend list` to track roadmap progress
7. Use `dust rollback` when recovery is needed
