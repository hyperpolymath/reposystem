== Claims Substantiation

=== Claim 1: "Visual wiring layer for multi-repo component management with aspect tagging and scenario comparison"

**How it works:** Reposystem scans local repository clones (`src/importers/`) to extract metadata (deps, APIs, names). The `src/graph/` module builds a directed graph where nodes are repositories and edges are dependencies or artifact flows. Each edge has a `channel` (artifact, runtime, source) and `evidence` pointing to CI config or Cargo.toml lines. The `src/aspects/` module overlays concern-based views (security, reliability, supply-chain) where each node/edge gets a weight. The TUI (`src/cli/`) and web UI (`web/`) render these graphs interactively; the web UI supports force-directed layout, ER mode, and accessibility (keyboard navigation, zoom, contrast).

**Caveat:** Graph discovery is file-pattern-based (parsing Cargo.toml, package.json) and may miss implicit dependencies. Aspect weights are manual annotations, not machine-learned; a security weight of "3/3" requires human judgment. For high-stakes decisions, pair with formal risk analysis.

**Evidence:** `src/importers/cargo.rs` and `src/importers/package.rs` parse dependency files; `src/graph/` defines node/edge/aspect data model; `web/graph.js` implements the interactive visualization.

=== Claim 2: "Scenario comparison shows how switching providers changes security, reliability, and supply-chain metrics"

**How it works:** The `src/scenarios/` module stores named configurations of the graph. A scenario is a snapshot: select which provider implementation each consumer uses for a given component (e.g., "use local podman" vs "use external docker"). The `scenario diff` command computes differences between two scenarios (how many edges changed, which nodes are affected) and re-evaluates aspect metrics on the new configuration. The `src/export/` module renders side-by-side comparisons showing before/after aspect scores (security 7/10 → 10/10, reliability 10/10 → 8/10).

**Caveat:** Aspect metric recalculation is deterministic but depends on accurate initial weights. If weights are unreliable, diffs are misleading. Integration with VeriSimDB or a formal risk database would improve fidelity.

**Evidence:** `src/scenarios/manager.rs` stores and diffs scenarios; `src/export/comparison.rs` computes metric deltas; the TUI shows side-by-side comparison view in the "SCENARIO COMPARISON" panel (see README mock UI).

== File Map

[cols="1,3"]
|===
| Path | Purpose

| `src/main.rs` | CLI entry: scan, list, view, export, group, edge, tag, scenario commands
| `src/importers/` | Repository discovery: Cargo.toml, package.json, go.mod parsers
| `src/importers/cargo.rs` | Extracts deps from Cargo.lock/Cargo.toml
| `src/importers/package.rs` | Extracts deps from package.json, yarn.lock, pnpm-lock.yaml

| `src/graph/model.rs` | Struct definitions: Graph, Node, Edge, Aspect, AspecTag, Weight
| `src/graph/validation.rs` | Detects circular dependencies, orphaned nodes, trust violations
| `src/aspects/` | Aspect tagging: security, reliability, supply-chain concern weights
| `src/aspects/tagger.rs` | Applies weights to nodes/edges based on aspect rules
| `src/scenarios/` | Scenario management: create, diff, apply, rollback
| `src/scenarios/manager.rs` | Stores scenarios, computes diffs, aggregates aspect deltas
| `src/export/` | Exporters: DOT, JSON, SVG, comparison reports
| `src/export/dot.rs` | Graphviz DOT output
| `src/export/json.rs` | JSON serialization for web UI

| `src/cli/` | Rust TUI: interactive graph view, group filtering, aspect overlays
