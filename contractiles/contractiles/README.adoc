This directory is the portable contract set. Copy it into any repository to establish a consistent operational, validation, trust, recovery, and intent framework.

== Format Policy

**A2ML is the canonical and preferred format for all contractile formulation.**

K9 Nickel components (`.k9.ncl`) serve as the validation and execution layer _above_ A2ML — they validate A2ML semantics via type contracts and provide Hunt-level automation, but they do not replace A2ML as the source of truth.

Legacy non-A2ML files (YAML Mustfile, Dustfile, Intentfile; Haskell Trustfile.hs) are deprecated and retained only for backward compatibility.

== The Five Contractile Types

[cols="1,2,2"]
|===
| Type | File | Purpose

| **Must** — Physical State
| `must/Mustfile.a2ml`
| What must be true (files, configs, invariants)

| **Trust** — Provenance State
| `trust/Trustfile.a2ml`
| What to verify (hashes, signatures, proofs)

| **Dust** — Reversibility State
| `dust/Dustfile.a2ml`
| How to recover (rollback, undo, replay)

| **Intend** — Intent State
| `lust/Intentfile.a2ml`
| What we commit to making true (roadmap)

| **K9** — Validated Components
| `k9/*.k9.ncl`
| Self-validating Nickel components (Kennel/Yard/Hunt)
|===

Each type has:

- A **canonical A2ML file** — the source of truth
- A **README** in its directory — quick-reference format guide
- A **full specification** — 600+ line `.adoc` with 5 golden examples
- A **K9 validator** — Yard-level Nickel contract for structural correctness

== Quick Start

[source,bash]
----
# Option 1: Scaffold from scratch
contractile init --name my-project

# Option 2: Copy from this directory
cp -r contractiles/ /path/to/my-project/contractiles/

# Then:
contractile gen-just                 # Generate Just recipes
must check                           # Verify Physical State
trust list                           # See available verifications
intend list                          # View roadmap
contractile status                   # Unified dashboard
----
