= TotalUpgrade
:toc: preamble
:icons: font

`total-upgrade` is a cross-platform Rust TUI meta-manager for tool and package management. It sits above `mise`, `asdf`, `opsm`, and system package managers to provide a unified dashboard for your entire environment.

== Overview
Managing multiple tool managers (asdf, mise, opsm) and system package managers can lead to fragmented environments. `total-upgrade` provides a single command-center to detect, sync, and discover tools across all your managers, ensuring consistency between Windows, WSL, and Linux.

== Features
* **Manager Detection:** Real-time detection of `asdf`, `mise`, and `opsm`.
* **Manifest Integration:** Unified view of tools managed by `.tool-versions`.
* **Interactive Toggling:** Select tools to sync or transfer between managers using the spacebar.
* **Proactive Discovery:** Scans for missing package managers when a runtime (Python, Node, etc.) is found.
* **Association Disambiguation:** Handles complex file associations like Coq vs V-lang `.v` files.
* **Background Daemon:** Automated checking for minor updates with advanced TUI configuration.
* **Standard Launcher:** Full system integration with Start Menu and Desktop shortcuts.

== Quick Start
To get started with `total-upgrade` on your system:

[source,bash]
----
# Build and integrate with your system
just integ

# Run the interactive TUI
total-upgrade-launcher --tui
----

Wondering how this works? See link:EXPLAINME.adoc[].

== Repository Layout
[cols="1,3", options="header"]
|===
| Path | Purpose
| `src/main.rs` | TUI entry point and state machine.
| `src/backend/` | Logic for detection, manifest parsing, and scanning.
| `launcher/` | Standard-compliant cross-platform launcher scripts.
| `manifest.scm` | GNU Guix environment definition.
| `Containerfile` | Podman containerization configuration.
|===
